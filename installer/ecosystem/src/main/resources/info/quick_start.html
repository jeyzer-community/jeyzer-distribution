Quick start

Take the owner's tour :
  • Choose a demo : Virtual threads, Labors (incident detection) or Features.
  • Run the demo and look at its Jeyzer recording
    • ${jeyzer.recorder.client.block.start}
  • Run the Jeyzer Client
    • ${jeyzer.recorder.client.block.end} ${jeyzer.analyzer.block.start}
  • Generate the JZR report
    • ${jeyzer.analyzer.block.end} ${jeyzer.monitor.block.start}
  • Monitor the demo application ${jeyzer.monitor.block.end}
  • Look at the demo profiles and customize it
  • Look at the demo monitoring rules and tune it
  • Look at the Base profiles and monitoring rules
    • ${jeyzer.recorder.agent.block.start}
  • Add the agent to your application
    • ${jeyzer.recorder.agent.block.end} ${jeyzer.analysis_or_monitor.block.start}
  • Create your own applicative profile ${jeyzer.analysis_or_monitor.block.end}

You can retrieve current info in quick_start.html at the root of the Jeyzer installation.

Recorder Agent :
Add this parameter to your application classpath :
-javaagent:"$INSTALL_PATH${SYSTEM[file.separator]}recorder${SYSTEM[file.separator]}lib${SYSTEM[file.separator]}jeyzer-agent.jar"=$INSTALL_PATH${SYSTEM[file.separator]}recorder${SYSTEM[file.separator]}config${SYSTEM[file.separator]}agent${SYSTEM[file.separator]}jeyzer-agent.xml;jeyzer-record-agent-profile=${jeyzer.recorder.target.profile}
The JZR recording will be available in this directory :
${jeyzer.recorder.work.dir}${SYSTEM[file.separator]}${jeyzer.recorder.target.profile}

Java Flight Recorder :
Add this standard JFR parameter to your application classpath :
--XX:StartFlightRecording=maxage=6h,maxsize=100M,dumponexit=true,settings=$INSTALL_PATH${SYSTEM[file.separator]}jfr${SYSTEM[file.separator]}jeyzer.jfc,filename=<JFR recording file path>
The jeyzer.jfc file is the recommended JFR configuration, aligned and optimized for Jeyzer.
